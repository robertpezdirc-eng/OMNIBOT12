KORAKI MIGRACIJE Z ANGEL AKTIVACIJO:

1. SISTEMSKA PRIPRAVA STREŽNIKA:
   - Namesti Ubuntu 22.04 LTS (ali ustrezen OS)
   - Posodobi sistem: apt update && apt upgrade -y
   - Namesti odvisnosti: python3, python3-pip, python3-venv, git, curl, wget
   - Namesti Nginx: apt install nginx -y
   - Namesti Certbot: apt install certbot python3-certbot-nginx -y
   - Namesti Node.js 18+: curl -fsSL https://deb.nodesource.com/setup_18.x | bash - && apt install nodejs -y
   - Konfiguriraj UFW firewall: ufw enable && ufw allow 22,80,443/tcp

2. MIGRACIJA OMNI APLIKACIJE:
   - Ustvari uporabnika: useradd -m -s /bin/bash omni
   - Kloniraj repozitorij: git clone https://github.com/your-repo/omniscient-ai-platform.git /opt/omni
   - Nastavi lastništvo: chown -R omni:omni /opt/omni
   - Ustvari virtualno okolje: cd /opt/omni && python3 -m venv .venv
   - Aktiviraj venv: source .venv/bin/activate
   - Namesti Python odvisnosti: pip install -r requirements.txt
   - Namesti Node.js odvisnosti: npm install
   - Testiraj aplikacijo: node omni-ultra-main.js (port 8080)

3. NGINX REVERSE PROXY KONFIGURACIJA:
   - Ustvari Nginx site: /etc/nginx/sites-available/omni
   - Konfiguriraj reverse proxy (8080 → 80/443)
   - Omogoči gzip kompresijo in varnostne headerje
   - Nastavi rate limiting: limit_req_zone $binary_remote_addr zone=omni:10m rate=10r/s
   - Ustvari simbolno povezavo: ln -s /etc/nginx/sites-available/omni /etc/nginx/sites-enabled/
   - Testiraj konfiguracijo: nginx -t
   - Reload Nginx: systemctl reload nginx

4. SSL CERTIFIKATI (Let's Encrypt):
   - Pridobi SSL certifikat: certbot --nginx -d moja-domena.com --email admin@moja-domena.com --agree-tos --non-interactive
   - Nastavi avtomatsko podaljševanje: systemctl enable certbot.timer
   - Testiraj podaljševanje: certbot renew --dry-run
   - Konfiguriraj HSTS in SSL optimizacije
   - Preusmeri HTTP na HTTPS

5. SYSTEMD STORITEV ZA OMNI:
   - Ustvari omni.service v /etc/systemd/system/
   - Konfiguriraj avtomatski zagon in restart ob napakah
   - Nastavi uporabnika 'omni' in delovni direktorij /opt/omni
   - Omogoči storitev: systemctl enable omni.service
   - Zaženi storitev: systemctl start omni.service
   - Preveri status: systemctl status omni.service
